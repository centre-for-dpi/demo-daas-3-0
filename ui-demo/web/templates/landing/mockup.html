Standards & Protocols

Active on this screen

W3C Verifiable Credentials Data Model 2.0 — the core standard defining how credentials are structured, signed, and verified.
The credential payload uses JSON-LD with a @context referencing the W3C VCDM 2.0 vocabulary. Proof is attached via Data Integrity (EdDSA/ECDSA) or secured externally via JOSE/COSE.

Issuance Protocol

OID4VCI — OpenID for Verifiable Credential Issuance.
The wallet sends a token request, receives an access token, then calls the credential endpoint with a proof of possession. Supports pre-authorized code flow (issuer-initiated) and authorization code flow (wallet-initiated).

Presentation Protocol

OID4VP — OpenID for Verifiable Presentations.
The verifier generates a presentation request encoded as a QR code or deep link. The holder's wallet parses the request, selects matching credentials, applies selective disclosure, and sends a Verifiable Presentation.

Status Check

Bitstring Status List — privacy-preserving credential revocation checking.
The issuer maintains a bitstring where each credential maps to a bit position. A bit value of 1 indicates revocation. The bitstring is compressed and cached for offline verification.

Proof Formats Supported

Five credential standards: W3C-VCDM 2.0, VCDM 1.1, SD-JWT, mDL (ISO 18013-5), and AnonCreds.
Each format has different selective disclosure capabilities. BBS+ and AnonCreds support zero-knowledge proofs. SD-JWT supports claim-level disclosure. mDL supports namespace-level disclosure.
