Pick a wallet backend first

Before you can claim a credential, you need to tell the platform where to store it. The five available wallet backends have genuinely different capabilities — one keeps credentials in a persistent cloud wallet (Walt.id), one runs as a browser-hosted wallet you pick credentials from directly (Inji Web), one is a quick in-memory test wallet (Local), one prints a self-verifying PDF (PDF Wallet), and one is a beta Sovrin agent (Credebl). The short wizard walks you through the choice.
