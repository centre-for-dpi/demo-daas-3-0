You're building an OID4VP verification request. Clicking Generate Request below asks walt.id's verifier-api to mint a fresh OID4VP authorization request — the standards-compliant way for a verifier to ask for a verifiable presentation. The next page renders that request as a copyable URL plus a QR code. Send either to a holder; their wallet picks a matching credential, builds a vp_token, and POSTs it back to walt.id's response_uri. The QR generator page then polls the session and shows the verified claims as soon as the holder responds.
This is the verifier-initiated path. The holder also has a sibling "Share Proactively" path on their Present Credential page that pushes a credential to a one-off /share/v/<id> URL — that's a platform convenience, not OID4VP. Use this page when you actually need a real OID4VP round-trip a wallet can respond to.
