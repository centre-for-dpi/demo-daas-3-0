Bulk Issuance

Issue many credentials at once. Active backend: {{$dpgName}}.

How bulk issuance works here: the three tabs below all do the same job — issue one credential at a time, repeated for every record. They just differ in where the records come from. None of the verifiable-credential backends we support today (walt.id, Inji Certify, Credebl) has a single-call "give me 1000 credentials" endpoint, so the platform always drives the loop on your behalf. Pick the tab that fits how your data is shaped.

Upload a spreadsheet (CSV)

Best for: a one-off batch you've already prepared in Excel or Google Sheets. Export it as CSV, drop it in below, tell us which column maps to which credential field, then click "Issue all". The platform reads each row and issues one credential per row.

The first row should be the column headers. We read it locally to preview the column names; the file is re-sent on submit.

Pull from a connected database

Best for: ongoing operations where the source of truth already lives in a system you run — a citizens registry in Postgres, a farmer database, an HR system, etc. The platform queries the database directly, pulls the rows, and issues one credential per row. No exports, no re-uploading every time the data changes.

Only databases that have been wired up to this deployment by an administrator show here.
We pull the first N rows from the database. Hard cap is 5000 per click — split larger jobs into multiple runs.
Pick "— skip —" for any credential field you'd rather leave blank.
{{end}}

Trigger from your own code (REST API)

Best for: embedding bulk issuance into a script, an n8n workflow, or your own backend. POST a JSON list of records to our endpoint and we issue one credential per record — exactly the same per-row work as the other two tabs, just driven from outside the UI. Use this when the data lives somewhere we can't reach directly, or when you want to schedule issuance from your existing automation.

POST /api/issuer/bulk-api 
curl -X POST http://localhost:8080/api/issuer/bulk-api \
  -H 'Content-Type: application/json' \
  -b cookies.txt \
  -d '{
    "configId": "UniversityDegree",
    "format": "ldp_vc",
    "records": [
      {"name":"Jelagat Gitau","degree":"BSc","major":"Computer Science"},
      {"name":"Wanjiru Kamau","degree":"MSc","major":"Medicine"}
    ]
  }'
{{end}}
